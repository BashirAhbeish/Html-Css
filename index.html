<picture> </picture>
Description:gives web developers more flexibility in specifying image resources.
The most common use of the element will be for art direction in responsive designs. Instead of having one image that is scaled up or down based on the viewport width, multiple images can be designed to more nicely fill the browser viewport.
The <picture> element contains two tags: <source> tags and one <img> tag.
The browser will look for the first <source> element where the media query matches the current viewport width, and then it will display the proper image (specified in the srcset attribute). The <img> element is required as the last child of the <picture> element, as a fallback option if none of the source tags matches.

Tip: The <picture> element works "similar" to <video> and <audio>. You set up different sources, and the first source that fits the preferences is the one being used.

 <table> </table>

An HTML table consists of one <table> element and one or more <tr>, <th>, and <td> elements.
An HTML table may also include <caption>, <colgroup>, <thead>, <tfoot>, and <tbody> elements.


<tbody> </tbody>
Description: used to group the body content in an HTML table.
Browsers can use these elements to enable scrolling of the table body independently of the header and footer. Also, when printing a large table that spans multiple pages, these elements can enable the table header and footer to be printed at the top and bottom of each page.
The <tbody> tag must be used in the following context: after any <caption>, <colgroup>, and <thead> elements.
parent: <table>
conjuncted with: <thead> , <tfoot>


<td> </td>
Description: defines a standard data cell in an HTML table.
An HTML table has two kinds of cells:
Header cells - (created with the <th> element)
Data cells - (created with the <td> element)
The text in <td> elements are regular and left-aligned by default.
attribute: colspan , headers , rowspan


<template> </template>
Description: used as a container to hold some HTML content hidden from the user when the page loads.
The content inside <template> can be rendered later with a JavaScript.
You can use the <template> tag if you have some HTML code you want to use over and over again, but not until you ask for it. To do this without the <template> tag, you have to create the HTML code with JavaScript to prevent the browser from rendering the code.


<textarea> </textarea>
Description: defines a multi-line text input control.
often used in a form, to collect user inputs like comments or reviews.
A text area can hold an unlimited number of characters, and the text renders in a fixed-width font (usually Courier).
The size of a text area is specified by the cols and rows attributes (or with CSS).
The name attribute is needed to reference the form data after the form is submitted (if you omit the name attribute, no data from the text area will be submitted).
The id attribute is needed to associate the text area with a label.
Tip: Always add the <label> tag for best accessibility practices!
attribute: autofocus , cols , dirname , disabled , form , maxlength , name , placeholder , readonly , required , rows , wrap


<tfoot> </tfoot>
Description: used to group footer content in an HTML table.
Browsers can use these elements to enable scrolling of the table body independently of the header and footer. Also, when printing a large table that spans multiple pages, these elements can enable the table header and footer to be printed at the top and bottom of each page.
The tag must be used after any <caption>, <colgroup>, <thead>, and <tbody> elements.
conjuncted with: <thead> , <tbody>


<th> </th>
Description: defines a header cell in an HTML table
An HTML table has two kinds of cells:
Header cells - (created with the <th> element)
Data cells - (created with the <td> element)
The text in <th> elements are bold and centered by default.
attribute: abbr , colspan , headers , rowspan , scope


<thead> </thead>
Description:used to group header content in an HTML table.
Browsers can use these elements to enable scrolling of the table body independently of the header and footer. Also, when printing a large table that spans multiple pages, these elements can enable the table header and footer to be printed at the top and bottom of each page.
The <thead> tag must be used after any <caption> and <colgroup> elements, and before any <tbody>, <tfoot>, and <tr> elements.
conjuncted with: <tbody> , <tfoot>


<time> </time>
Description: defines a specific time (or datetime).
The datetime attribute of this element is used translate the time into a machine-readable format so that browsers can offer to add date reminders through the user's calendar, and search engines can produce smarter search results.
attribute: datetime


<title> </title>
Description: defines the title of the document. The title must be text-only, and it is shown in the browser's title bar or in the page's tab.
The <title> tag is required in HTML documents!
The contents of a page title is very important for search engine optimization (SEO)! The page title is used by search engine algorithms to decide the order when listing pages in search results.
Here are some tips for creating good titles:,
  • Go for a longer, descriptive title (avoid one- or two-word titles)
  • Search engines will display about 50-60 characters of the title, so try not to have titles longer than that
  • Do not use just a list of words as the title (this may reduce the page's position in search results)
So, try to make the title as accurate and meaningful as possible!
parent: <head>


<tr> </tr>
Description: defines a row in an HTML table.
contains one or more <th> or <td> elements.


<track> </track>
Description: specifies text tracks for <audio> or <video> elements.
This element is used to specify subtitles, caption files or other files containing text, that should be visible when the media is playing.
Tracks are formatted in WebVTT format (.vtt files).
optional attribute default , kind , label , src , srclang

<video> </video>
Description: used to embed video content in a document, such as a movie clip or other video streams.
contains one or more <source> tags with different video sources. The browser will choose the first source it supports.
The text between the <video> and </video> tags will only be displayed in browsers that do not support the <video> element.
There are three supported video formats in HTML: MP4, WebM, and OGG.
attributes: autoplay , controls , height , loop , muted , poster , preload , src , width

Block (Flow) Elements


Description: start on a new line, taking up the full width available, and the browsers automatically add some space (a margin) before and after the element.
They always create a new block of content, which means elements following them will appear on the next line.
respect all margin and padding settings, meaning they can push other elements away both horizontally and vertically.
They can contain other block-level or inline elements, making them useful for creating complex layouts.

Inline (Phrasing) Elements


Description:occupy the space as needed within the space defined by the main element.
they do not begin on new lines.
can have margins and paddings applied, but these won't affect the layout in the same way as block-level elements. Specifically, vertical margins and padding won't push other elements away.
They can be styled using CSS, but their inline nature limits certain layout properties.
